Javascript does work very efficient ways almost all modern browsers. Whenever any javascript code needs to execute, it might be browser or run time like nodeJs, javascript followed by a set of rules, for instance, synchronous neither asynchronous. To execute Javascript code, run time in which is nodeJs use V8 engine that's the opensource of google chrome browser engine. Simply three steps are being followed by execution context of Javascript code, in-depth run time also more functional task handle dynamically, three steps are -
step-01: Source code to syntax tree(Browser generate AST)
Step-02: AST sent the code into the interpreter to convert in bytecode.
Step-03: Bytecode to machine code.
 Following the process, Javascript rapidly responds to the user interaction compared to the other competitive language.

Nowadays, the most popular programming language is Javascript. The language is getting more popularity among the developers' domain due to some exclusive features, for example, the Event loop. Javascript is commonly known as a single thread language as if it works with a single-thread feature, it has followed by a synchronous and asynchronous approach. During the run time, the JS execution context handle very organised ways. When the Code call to execute synchronous behaviour follow the LIFO(Last-in, first-out) approach algorithm that is called a Stack. More importantly, during the execution phase any present interval task which needs a tiny more time than synchronous task the JS run time directly send the code its assistance, like Web API, at the same time web assistant help to execute the task and place it on the callback queue, after finishing the synchronous task from Stack JS execute the callback queue task, here is the significant role player is Event Loop, that always monitoring the Stack task, whenever the task will finish event loop working to pass the task callback queue to stack. The event loop is an intermediator of the callback queue to the Stack.

LocalStorage and SessionStroage are both used in browsers for some common purposes yet, the use-case might differ considering the application requirements. Here is some differences -
1. LocalStorage has no expiration date but SessionStorage for short time until close the tab.
2. LocalStorage data can not send to the server but SessionStorage data can.
3. LocalStorage capacity of data capture is relatively more than sessionStorage.
4. LocalStorage data need manually delete or algorithm but sessionStorage data automatic delete.
5. LocalStorage can use in all windows but SessionStorage uses a private tab.
